    Organizing Group Vars Files in Ansible

    By Osvaldo Toja September 14, 2014

    from here

    In addition to the storing variables directly in the INI file, host and group variables can be stored in individual files relative to the inventory file.

    directory layout

     1production/
 2├── group_vars
 3│   └── server.yml
 4└── inventory
 5staging/
 6├── group_vars
 7│   └── server
 8└── inventory
 9group-vars.yml

    playbook

    1# group-vars.yml
2- hosts: all
3  user: osvaldo
4  sudo: no
5  gather_facts: False
6
7  tasks:
8     - debug: msg="reading from "

    files in production directory

    inventory

    1# production/inventory
2[server]
3localhost   ansible_connection=local

    vars

    1# production/group_vars/server.yml
2env_name: production

    files in staging directory

    inventory

    1# staging/inventory
2[server]
3localhost   ansible_connection=local

    vars

    1# staging/group_vars/server
2env_name: staging

    executing the playbook

    using data from staging directory

     1$ ansible-playbook -i staging group-vars.yml
 2
 3PLAY [all] ********************************************************************
 4
 5TASK: [debug msg="reading from "] *********************************
 6ok: [localhost] => {
 7    "msg": "reading from staging"
 8}
 9
10PLAY RECAP ********************************************************************
11localhost                  : ok=1    changed=0    unreachable=0    failed=0

    using data from production directory

     1$ ansible-playbook -i production group-vars.yml
 2
 3PLAY [all] ********************************************************************
 4
 5TASK: [debug msg="reading from "] *********************************
 6ok: [localhost] => {
 7    "msg": "reading from production"
 8}
 9
10PLAY RECAP ********************************************************************
11localhost                  : ok=1    changed=0    unreachable=0    failed=0

    Interesting fact: ansible-playbook when provided a directory as the inventory, will search by default a file named inventory so no need to specify -i production/inventory, only -i production will work just fine.
